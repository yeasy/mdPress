# Output Formats

mdPress can generate your documentation in multiple formats, each optimized for different use cases. You can build a single format or generate multiple formats simultaneously with a single command.

## Site Output

The site format generates a modern, multi-page static website with client-side navigation and advanced features.

### Site Features

The site output includes:

- **Multi-page structure**: Each chapter becomes a separate HTML page for fast loading
- **SPA navigation**: Single-Page Application style navigation without full page reloads
- **Full-text search**: Client-side search across all content with keyboard shortcut (Cmd/Ctrl+K)
- **Dark mode**: Automatic dark/light theme switching with user preference persistence
- **Responsive design**: Mobile-friendly layout that adapts to all screen sizes
- **Table of contents**: Automatic sidebar with chapter hierarchy and current page highlighting
- **Social sharing**: Built-in metadata for sharing on social platforms

### Building a Site

Generate a site with the default configuration:

```bash
mdpress build --format site
```

Specify an output directory:

```bash
mdpress build --format site --output ./public
```

The output directory structure will be:

```
public/
├── index.html
├── chapter-1/
│   └── index.html
├── chapter-2/
│   └── index.html
├── assets/
│   ├── style.css
│   ├── script.js
│   └── images/
├── search-index.json
└── sitemap.xml
```

### SPA Navigation

The site uses a single-page application architecture. When users navigate between chapters:

1. Only the content changes, not the page structure
2. No full page reload occurs
3. Scroll position is automatically managed
4. Browser history is maintained for back/forward navigation

This provides a fast, smooth experience similar to native applications.

### Search Functionality

The site includes client-side full-text search powered by a search index. Users can:

- Press Cmd+K (Mac) or Ctrl+K (Windows/Linux) to open the search dialog
- Search across all chapters and sections
- Click results to jump to relevant sections
- See highlighted search terms in the content

The search index is generated automatically during the build process.

### Dark Mode

Users can toggle between light and dark themes. The preference is automatically saved in browser local storage and persists across sessions. You can customize theme colors in your `book.yaml` configuration.

## PDF Output

Generate high-quality PDF documents suitable for printing and sharing.

### PDF Generation Methods

mdPress supports two PDF generation engines:

#### Chromium-based PDF

The default method using headless Chromium:

```bash
mdpress build --format pdf
```

Features:
- High fidelity rendering matching the browser display
- Support for CSS print styles
- Fast generation
- Suitable for most documentation

#### Typst-based PDF

An alternative using the Typst typesetting engine:

```bash
mdpress build --format pdf --engine typst
```

Features:
- Professional typography
- Better control over layout
- Smaller file sizes
- Better performance on large documents

### PDF Configuration

Configure PDF output in your `book.yaml`:

```yaml
pdf:
  engine: "chromium"  # or "typst"
  paperSize: "A4"
  margin:
    top: 20
    bottom: 20
    left: 15
    right: 15
  headerTemplate: "<span class='title'></span>"
  footerTemplate: "<span class='page'></span> of <span class='pageCount'></span>"
  printBackground: true
```

### PDF Features

PDFs generated by mdPress include:

- **Bookmarks**: Table of contents creates clickable PDF bookmarks for navigation
- **Watermarks**: Optional watermark text for draft versions or confidential documents
- **Headers and footers**: Automatic page numbers and custom text
- **Embedded fonts**: Professional font rendering
- **Hyperlinks**: Internal links and external URLs remain clickable
- **Images and diagrams**: Full support for images, Mermaid, and PlantUML diagrams

### Adding Watermarks

Configure watermarks in `book.yaml`:

```yaml
pdf:
  watermark: "DRAFT"
  watermarkColor: "rgba(200, 200, 200, 0.5)"
```

Or use the command line:

```bash
mdpress build --format pdf --watermark "CONFIDENTIAL"
```

## HTML Output

Generate a single self-contained HTML file with all content and styling embedded.

### Creating Single-File HTML

```bash
mdpress build --format html
```

The output is a single `index.html` file that contains:

- All content and structure
- Embedded CSS styles
- Embedded JavaScript for interactivity
- Base64-encoded images
- No external dependencies

### Use Cases for Single-File HTML

Single-file HTML is useful for:

- Sharing documentation via email
- Archiving documentation as a complete unit
- Offline distribution
- Printing to paper or PDF from a browser
- Systems that require zero external dependencies

### HTML Features

Single-file HTML output includes:

- Full navigation in a sidebar
- Search functionality (with data embedded in the file)
- Dark mode support
- Responsive design
- All interactive features of the site format

### File Size Considerations

Single-file HTML includes everything, which can result in larger files. For documentation with many images or large diagrams, consider:

- Compressing the HTML file
- Using site format for web delivery
- Splitting very large documentation into multiple HTML files

## ePub Output

Generate ePub files for e-readers like Kindle, Apple Books, and other devices.

### Creating ePub

```bash
mdpress build --format epub
```

The output is an `index.epub` file containing:

- Standard ePub 3.0 format
- All chapters as separate documents
- Table of contents with navigation
- Embedded fonts and images
- Support for reflowable text

### ePub Features

ePub files support:

- **Reflowable layout**: Text adapts to reader screen size
- **Table of contents**: Clickable navigation
- **Bookmarks**: Reader can save progress
- **Full-text search**: Built into most e-readers
- **Code blocks**: Displayed with monospace fonts
- **Mathematical equations**: Rendered as images in most e-readers
- **Images and diagrams**: Embedded and displayed

### ePub Configuration

Configure ePub output in `book.yaml`:

```yaml
epub:
  language: "en"
  author: "Your Name"
  publisher: "Your Organization"
  cover: "./assets/cover.png"
```

### ePub Limitations

Some features have limitations in ePub format:

- Interactive elements are converted to static content
- Complex CSS layouts may not render identically
- JavaScript functionality is not supported
- Some diagram types may render as images

## Building Multiple Formats

Generate multiple output formats with a single command.

### Build All Formats

```bash
mdpress build --format all
```

This generates: site, PDF, HTML, and ePub.

### Build Specific Formats

```bash
mdpress build --format pdf,html,epub
```

Comma-separated format names generate only the specified formats.

### Parallel Building

When building multiple formats, mdPress:

1. Parses your documentation once
2. Processes common transformations
3. Applies format-specific rendering in parallel
4. Outputs all formats simultaneously

This is faster than building each format separately.

### Building Multiple Formats

Example command for common scenarios:

```bash
# Web and PDF for distribution
mdpress build --format site,pdf --output ./dist

# All formats for comprehensive distribution
mdpress build --format all --output ./dist

# Only PDF for printing
mdpress build --format pdf --output ./print
```

### Output Organization

When building multiple formats to the same directory:

```
dist/
├── site/
│   ├── index.html
│   ├── chapter-1/
│   ├── assets/
│   └── search-index.json
├── index.html (single-file HTML)
├── index.epub
├── index.pdf
└── build-report.json
```

## Output Customization

### Customizing the Site

Modify `book.yaml` to customize site appearance:

```yaml
title: "My Documentation"
description: "Complete guide to my project"

site:
  baseUrl: "https://docs.example.com"
  theme:
    colorScheme: "light"  # or "dark" or "auto"
    primaryColor: "#2563eb"
    accentColor: "#dc2626"
  footer: "Copyright 2026 Example Corp"
  logo: "./assets/logo.svg"
```

### Asset Management

Place static assets in an `assets` directory:

```
book/
├── book.yaml
├── SUMMARY.md
├── chapter-1.md
├── chapter-2.md
└── assets/
    ├── logo.svg
    ├── favicon.ico
    └── images/
        ├── screenshot-1.png
        └── diagram.svg
```

Assets are automatically copied to the output directory and made available in the `assets/` path.

## Deployment Considerations

### Site Deployment

Site format is ideal for web deployment:

```bash
# Build for production
mdpress build --format site --output ./public

# Deploy to web server
rsync -av public/ user@server:/var/www/docs/
```

### PDF Distribution

PDFs work well for email and downloads:

```bash
# Generate with specific name
mdpress build --format pdf --output "./dist/Manual-v1.0.pdf"
```

### Static Hosting

The site format requires only static file hosting:

- GitHub Pages
- Netlify
- Vercel
- Any static host (AWS S3, Cloudflare Pages, etc.)

No server-side processing required.

## Troubleshooting

### PDF Generation Fails

- Ensure Chromium is installed if using the default engine
- Check available disk space
- Try the Typst engine if Chromium has issues
- Review error messages for specific missing dependencies

### Large File Sizes

- Consider compression for single-file HTML output
- Split very large documentation
- Optimize images before including them
- Use site format for web distribution instead of single HTML file

### Search Not Working

- Verify search index is generated in the build output
- Check browser console for JavaScript errors
- Ensure JavaScript is enabled in target environment
- For single-file HTML, test in different browsers

### Rendering Issues in Specific Formats

- Test problematic content in different output formats
- Some features (like interactive elements) don't work in all formats
- Simplify complex CSS for PDF output
- Use fallbacks for advanced features in ePub format
