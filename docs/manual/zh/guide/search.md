# 搜索功能

mdPress 包含完全在浏览器中运行的内置全文搜索，无需后端服务器。本指南解释了搜索如何工作以及如何有效地使用它。

## 搜索如何工作

mdPress 中的搜索功能是客户端的，这意味着所有搜索操作都在你的浏览器中发生，无需联系服务器。

### 搜索索引

构建文档时，mdPress 生成一个名为 `search-index.json` 的搜索索引文件。此文件包含：

- 文档中的所有标题和子标题
- 每个部分的完整文本内容
- 页面元数据（标题、URL、章节层次）
- 用于相关性排名的词频数据

搜索索引自动包含在你的网站和单文件 HTML 输出中。

### 索引生成

搜索索引在构建过程中创建：

```bash
mdpress build --format site
```

mdPress 自动：
1. 从你的 Markdown 提取所有文本内容
2. 赋予标题比正文更高的索引权重
3. 为每个部分创建入口点
4. 压缩索引以实现快速加载
5. 在你的输出文件中嵌入索引

### 索引大小

搜索索引大小取决于你的文档：
- 小文档（10-50 页）：100-300KB
- 中等文档（50-200 页）：300KB-1MB
- 大文档（200+ 页）：1-5MB

索引仅在需要时加载并在浏览器中缓存。

## 使用搜索功能

用户可以根据输出格式以多种方式访问搜索。

### 打开搜索面板

按键盘快捷方式打开搜索面板：

- **Mac**：Cmd+K
- **Windows/Linux**：Ctrl+K

或者，点击文档界面中的搜索按钮（如果可见）。

### 执行搜索

1. 用 Cmd/Ctrl+K 打开搜索面板
2. 输入你的搜索查询
3. 结果在你输入时实时出现
4. 点击结果跳转到该部分
5. 按 Escape 关闭搜索面板

### 搜索查询语法

基本搜索使用简单的关键字匹配：

```
api documentation
```

按相关性顺序搜索"api"和"documentation"。

多个术语：
```
REST API authentication
```

返回包含所有三个术语的结果，按相关性排序。

引用的短语：
```
"authentication token"
```

搜索确切短语"authentication token"。

### 搜索结果

结果显示为：

- **标题**：匹配发生的部分或标题
- **预览**：显示上下文中匹配的文本片段
- **突出显示**：搜索词在预览中突出显示
- **分数**：相关性排名（越高越好）
- **面包屑**：显示结果位置的章节层次

## 搜索结果突出显示

当你导航到搜索结果时，页面自动：

1. 滚动到匹配的部分
2. 在内容中突出显示搜索词
3. 更新浏览器 URL 以指向特定部分
4. 更新目录以显示你的当前位置

突出显示是临时的，当你与页面交互时淡出。

## 中日韩 (CJK) 搜索支持

mdPress 支持中文、日文和韩文的搜索。

### CJK 分词

对于 CJK 语言（中文、日文、韩文），mdPress 使用专门的分词：

- 字符单独索引，因为这些语言不使用空格
- 每个字符可以独立匹配
- 搜索结果包括所有包含搜索字符的内容
- 排名考虑字符接近度和频率

### CJK 搜索示例

中文搜索：
```
API文档
```

返回包含"API"或"文档"的结果。

日文搜索：
```
認証トークン
```

返回具有匹配字符的结果。

韩文搜索：
```
인증 토큰
```

返回具有匹配字符和单词的结果。

### 混合语言搜索

跨混合语言内容搜索：
```
API 文档 authentication
```

找到包含英文和 CJK 术语的结果。

## 不同输出格式中的搜索

### 网站格式

具有完整搜索功能：
- 输入时实时搜索
- 多个结果突出显示
- 功能完整的搜索面板
- 键盘快捷方式

### 单文件 HTML

搜索与网站格式相同：
- 索引嵌入在 HTML 文件中
- 无需外部请求
- 完全离线工作
- 所有功能可用

### PDF 格式

搜索由 PDF 阅读器提供：
- 使用 PDF 查看器的搜索功能 (Cmd/Ctrl+F)
- 无 mdPress 特定搜索索引
- 搜索功能取决于 PDF 查看器

### ePub 格式

搜索取决于电子阅读器应用程序：
- 使用电子阅读器的内置搜索
- 质量取决于设备
- 在设备上离线工作

## 性能和优化

### 索引的延迟加载

搜索索引是延迟加载的，这意味着：
- 在用户打开搜索之前不加载
- 首次搜索可能需要 100-500ms 加载索引
- 后续搜索是即时的
- 索引为会话缓存

### 搜索响应时间

典型搜索响应时间：
- 构建时索引：大多数项目 1-5 秒
- 首次搜索（冷）：100-500ms
- 后续搜索（热）：10-50ms
- 显示结果：<100ms

## 故障排除

### 搜索不工作

验证：
1. 你的浏览器中启用了 JavaScript
2. 搜索索引文件存在（在浏览器 DevTools 网络选项卡中检查）
3. 搜索面板打开（尝试 Cmd/Ctrl+K）
4. 在不同的浏览器中尝试

如果搜索仍不工作，重建你的文档：
```bash
mdpress build --format site
```

### 缓慢的搜索性能

对于大型文档（500+ 页）：
1. 在配置中限制最大结果
2. 通过排除部分来减少索引大小
3. 清除浏览器缓存并重新加载
4. 使用现代浏览器（Chrome、Firefox、Safari、Edge）

### 不正确的搜索结果

如果搜索结果看起来错误：
1. 验证搜索索引在最新构建中重新生成
2. 检查内容在你的 Markdown 中正确格式化
3. 搜索不同的关键字
4. 使用 `--no-cache` 标志强制完整重建

### CJK 搜索问题

如果 CJK 搜索不工作：
1. 确保内容编码为 UTF-8
2. 检查浏览器控制台的错误
3. 尝试用原始语言搜索，而不是翻译文本

## SEO 和搜索引擎

mdPress 搜索功能用于文档内搜索，而非网页搜索引擎优化。

### 外部搜索索引

帮助 Google 等外部搜索引擎索引你的文档：

1. 确保你的网站有站点地图（自动生成 `sitemap.xml`）
2. 在 `book.yaml` 配置中添加适当的元标签
3. 向 Google Search Console 提交你的站点地图
4. 确保你的托管允许搜索引擎爬虫

## 最佳实践

### 为更好的搜索组织结构

编写清晰的标题和描述性的部分标题：

```markdown
# 配置 API 认证

## Bearer Token 认证

## OAuth 2.0 集成
```

这创建比以下更好的搜索结果：

```markdown
# 设置

## 选项 1

## 选项 2
```

### 使用一致的术语

在文档中始终一致地使用相同的术语。例如，始终使用"认证"而不是混合"auth"、"login"和"sign-in"。

### 包括常见的同义词

在内容中提及用户可能搜索的替代术语：

```markdown
# 认证（登录，注册）

用户可以认证、登录或注册到他们的账户。
```

这帮助搜索找到内容，即使用户搜索同义词。

### 添加搜索友好的元数据

使用 `GLOSSARY.md` 文件来定义术语及其描述。每个术语是一个二级标题，后面跟着其定义：

```markdown
## API 密钥
用于验证 API 请求的唯一令牌。

## 速率限制
每个时间段内允许的最大 API 调用次数。
```

词汇表中的术语会在内容中自动高亮和链接，同时也能提高搜索的可发现性。
