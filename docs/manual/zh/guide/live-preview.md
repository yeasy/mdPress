# 实时预览服务器

实时预览服务器让你在编写时看到文档实时更新。本指南涵盖了如何使用 mdPress serve 命令和理解实时重新加载的工作原理。

## 启动预览服务器

使用单个命令启动实时预览服务器：

```bash
mdpress serve
```

这会启动一个本地网络服务器并自动在默认浏览器中打开你的文档。默认 URL 是 `http://localhost:9000`。

## 服务器命令选项

`serve` 命令接受多个标志来自定义预览环境。

### 指定端口

如果 9000 已在使用，更改端口：

```bash
mdpress serve --port 3000
```

你的文档将在 `http://localhost:3000` 上可用。

### 指定主机

允许来自网络上其他机器的访问：

```bash
mdpress serve --host 0.0.0.0
```

现在你可以使用你的机器 IP 地址从另一台计算机访问服务器，如 `http://192.168.1.100:9000`。

### 自动打开浏览器

默认情况下，mdPress 自动打开浏览器。禁用此行为：

```bash
mdpress serve --no-open
```

在远程系统上或开发环境中有用，你在其中单独管理浏览器。

### 自定义输出目录

从特定输出目录提供服务：

```bash
mdpress serve --output ./public
```

如果你已经构建了文档并想预览构建的输出而不是从源文件实时构建，这很有用。

## 完整的服务器配置

结合多个标志满足你的特定设置：

```bash
mdpress serve --port 8080 --host 0.0.0.0 --no-open --output ./build
```

这在端口 8080 上启动一个服务器，可从任何网络接口访问，不打开浏览器，从 `./build` 目录提供预构建的文件。

## 文件监视和自动重建

mdPress 自动监视源文件的更改并重建受影响的内容。

### 监视的文件

服务器监视：

- Markdown 文件（书籍目录中的 `.md` 文件）
- 配置文件（`book.yaml`、`LANGS.md`）
- 资源文件（图像、样式表、自定义脚本）
- SUMMARY.md（目录）
- 语言配置文件（在多语言设置中）

### 重建过程

保存文件时：

1. mdPress 在 100ms 内检测到更改
2. 相关文件被重新解析和重新渲染
3. 更新的内容为提供准备就绪
4. 实时重新加载信号发送到浏览器
5. 浏览器原子地更新内容
6. 你的视图更新而不丢失滚动位置或表单状态

### 增量构建

对于大型文档项目，mdPress 使用增量构建：

- 仅重建修改的章节
- 未更改的内容重用之前的构建结果
- 共享资源仅构建一次
- 搜索索引增量更新

即使对于大型项目，这也保持了快速的重建时间。

## 基于 WebSocket 的实时重新加载

实时重新加载机制使用 WebSocket 连接进行即时更新。

### 实时重新加载如何工作

1. **初始连接**：浏览器在页面加载时连接到开发服务器
2. **文件监视**：服务器持续监视源文件
3. **更改检测**：当文件更改时，服务器检测到它
4. **WebSocket 消息**：服务器通过 WebSocket 发送重新加载消息
5. **内容更新**：浏览器接收消息并刷新内容
6. **原子交换**：新内容以单个操作原子地替换旧内容

### 原子交换机制

原子交换确保平滑更新：

- 新 HTML 在任何页面更改之前在内存中准备好
- CSS 和 JavaScript 无声地加载
- 旧内容被删除，新内容以单个操作插入
- 用户看不到中间状态
- 尽可能保留滚动位置

这可防止闪烁并维护平滑的预览体验。

### 网络弹性

WebSocket 连接具有弹性：

- 如果连接断开，自动重新连接
- 状态指示器显示连接状态（通常在连接时隐藏）
- 如果服务器重启，浏览器自动重新连接
- 如果 WebSocket 不可用，优雅降级

## 开发工作流

### 典型的实时预览会话

1. 启动服务器：
   ```bash
   mdpress serve
   ```

2. 文档自动在你的浏览器中打开

3. 编辑你的 Markdown 文件：
   ```bash
   vim docs/chapter-1.md
   ```

4. 保存文件 - 预览立即更新

5. 继续编辑和预览

### 多显示器设置

在一个显示器上使用实时预览，同时在另一个显示器上编辑：

```bash
mdpress serve --port 9000 --host 0.0.0.0
```

在第二个显示器或设备上查看文档，同时在文本编辑器中编辑。

### 跨设备测试

从网络上的多个设备访问预览：

```bash
mdpress serve --host 0.0.0.0 --port 9000
```

然后从以下位置导航到 `http://<your-machine-ip>:9000`：
- 手机：`http://192.168.1.100:9000`
- 平板电脑：`http://192.168.1.100:9000`
- 其他计算机：`http://192.168.1.100:9000`

所有设备实时看到相同的实时更新。

## 性能考虑

### 重建时间

对于大多数文档：
- 小更改：50-100ms
- 完整重建：200-500ms
- 大型项目（1000+ 页）：1-2 秒

时间取决于你的硬件和项目复杂性。

### 内存使用

实时预览服务器：
- 在内存中保持源文件以快速访问
- 维护当前构建状态
- 为撤销/重做操作存储一级历史
- 典型内存使用：大多数项目 100-500MB

对于非常大的项目，你可以重启服务器以释放内存。

### 禁用自动重建

如果自动重建影响性能，禁用它：

```bash
mdpress serve --no-watch
```

然后手动触发重建（尽管此功能可能因配置而异）。

## 故障排除

### 端口已在使用

如果端口 9000 已在使用，指定不同的端口：

```bash
mdpress serve --port 8080
```

或查找使用该端口的进程：

```bash
lsof -i :9000        # macOS/Linux
netstat -ano | grep 9000  # Windows
```

### 更改未反映

确保你编辑的文件在监视目录中：
- 源 Markdown 文件必须在你的书籍目录中
- 配置必须在根目录中
- 资源必须在资源文件夹中

如果仍不工作，尝试重启服务器：

```bash
# 关闭当前服务器并重启
mdpress serve
```

### 重建时间缓慢

对于大型项目，可以减少重建缓慢：

1. 检查系统资源（CPU、磁盘空间）
2. 从监视中排除大型二进制资源
3. 如果可能，将文档分割成多个项目
4. 使用 `--output` 标志的预构建输出模式

### 浏览器未连接

如果浏览器选项卡显示"等待连接"：

1. 检查服务器是否运行
2. 验证正确的 URL（应匹配你指定的端口）
3. 检查 WebSocket 是否被防火墙阻止
4. 尝试从不同的浏览器访问
5. 重启服务器和浏览器

### 实时重新加载不工作

WebSocket 问题可以阻止实时重新加载：

1. 检查浏览器控制台的连接错误
2. 确保你的代理/防火墙不阻止 WebSocket 连接
3. 尝试通过 IP 地址而不是 localhost 访问
4. 重启开发服务器

## 高级用法

### 提供多个项目

在不同端口上运行单独的服务器：

```bash
# 终端 1
cd project-a
mdpress serve --port 9000

# 终端 2
cd project-b
mdpress serve --port 9001
```

在 `http://localhost:9000` 和 `http://localhost:9001` 访问两者。

### 与 IDE 集成

大多数现代 IDE 可以运行终端命令。设置你的 IDE 运行 `mdpress serve` 并在侧面板或单独的窗口中查看预览。

### 环境变量

某些配置可以通过环境变量设置：

```bash
export MDPRESS_PORT=8080
export MDPRESS_HOST=0.0.0.0
mdpress serve
```

### 构建配置

实时预览尊重你的 `book.yaml` 配置：
- 目录结构
- 主题设置
- 语言配置
- 自定义脚本和样式
- Markdown 扩展

对 `book.yaml` 的更改会触发预览的完整重建。
