# 扩展 Markdown 功能

mdPress 通过强大的功能扩展标准 Markdown，包括数学方程、图表、脚注和自动链接。本指南涵盖 mdPress 中可用的高级 Markdown 功能。

## KaTeX 数学表达式

使用 KaTeX 语法编写数学方程。支持内联和展示模式。

### 内联数学

使用单个美元符号编写内联数学表达式：

```markdown
二次公式是 $ax^2 + bx + c = 0$。

爱因斯坦著名的方程是 $E = mc^2$。

前 $n$ 个自然数的和是 $\frac{n(n+1)}{2}$。
```

### 展示数学

使用双美元符号显示在自己的行上渲染的方程：

```markdown
勾股定理：

$$a^2 + b^2 = c^2$$

圆的公式：

$$A = \pi r^2$$
```

### 复杂方程

mdPress 支持所有标准 KaTeX 函数和符号：

```markdown
$$\int_0^\infty e^{-x^2} dx = \frac{\sqrt{\pi}}{2}$$

$$\frac{\partial f}{\partial x} = \lim_{h \to 0} \frac{f(x+h) - f(x)}{h}$$

$$\begin{pmatrix} a & b \\ c & d \end{pmatrix} \begin{pmatrix} x \\ y \end{pmatrix} = \begin{pmatrix} ax + by \\ cx + dy \end{pmatrix}$$
```

### 数学最佳实践

- 为重要方程使用展示模式 (`$$`)
- 为文本内的简短表达式使用内联模式 (`$`)
- 在复杂方程前添加解释文本
- 在实时预览中测试渲染以确保正确输出

## Mermaid 图表

使用 Mermaid 语法创建流程图、序列图、状态图等。

### 流程图

可视化流程和决策树：

```markdown
​```mermaid
flowchart TD
    A[开始] --> B{决策}
    B -->|是| C[流程 A]
    B -->|否| D[流程 B]
    C --> E[结束]
    D --> E
​```
```

### 序列图

显示系统或参与者之间的交互：

```markdown
​```mermaid
sequenceDiagram
    actor 用户
    用户->>浏览器: 点击按钮
    浏览器->>服务器: 发送请求
    服务器->>数据库: 查询数据
    数据库-->>服务器: 返回结果
    服务器-->>浏览器: 发送响应
    浏览器-->>用户: 显示结果
​```
```

### 状态图

文档化状态机和工作流：

```markdown
​```mermaid
stateDiagram-v2
    [*] --> 空闲
    空闲 --> 加载: 用户点击
    加载 --> 准备: 数据已加载
    准备 --> 处理: 点击开始按钮
    处理 --> 完成: 完成
    完成 --> [*]
​```
```

### 类图

模型化面向对象的系统：

```markdown
​```mermaid
classDiagram
    class 动物 {
        -String 名称
        +String 获取名称()
        +void 设置名称(String)
    }
    class 狗 {
        +void 叫()
    }
    动物 <|-- 狗
​```
```

### 甘特图

使用时间线可视化进行项目规划：

```markdown
​```mermaid
gantt
    title 项目时间线
    section 设计
    线框图 :d1, 2026-03-01, 10d
    原型图 :d2, after d1, 10d
    section 开发
    前端 :d3, 2026-03-15, 20d
    后端 :d4, 2026-03-20, 25d
    section 测试
    质量保证 :d5, after d3, 10d
​```
```

### 饼图

显示数据分布：

```markdown
​```mermaid
pie title 浏览器市场份额
    "Chrome" : 65
    "Firefox" : 15
    "Safari" : 15
    "其他" : 5
​```
```

### 支持的图表类型

mdPress 中的 Mermaid 图表支持：流程图、序列图、状态图、类图、实体关系图、git 图、用户旅程和饼图。根据你的 mdPress 版本，可能还支持其他图表类型。

## PlantUML 图表

使用 PlantUML 语法创建复杂图表，作为 Mermaid 的替代：

```markdown
​```plantuml
@startuml
actor 用户
用户 -> 网络应用: 点击按钮
网络应用 -> 服务器: 发送请求
服务器 -> 数据库: 查询
数据库 --> 服务器: 返回数据
服务器 --> 网络应用: 响应
网络应用 --> 用户: 显示
@enduml
​```
```

### PlantUML 图表类型

支持的图表类型包括：用例、序列、类、状态、活动、组件、部署、对象和定时图表。

### 何时使用 PlantUML vs Mermaid

为以下情况使用 **Mermaid**：
- 快速流程图和决策树
- 简单序列图
- 甘特项目时间线
- 数据可视化

为以下情况使用 **PlantUML**：
- 复杂 UML 图表
- 详细的类层次结构
- 组件和部署图表
- 需要精确控制时

## 脚注

添加脚注以提供额外上下文，而不中断主要文本：

```markdown
这是带脚注的一些文本[^1]。

带有不同脚注的另一句话[^2]。

[^1]: 这是第一个脚注的内容。
[^2]: 这是第二个脚注。如果缩进
    连续行，它可以跨越多行。
```

### 脚注功能

- 脚注自动编号
- 读者可以点击脚注引用跳转到脚注
- 脚注出现在文档末尾
- 对于有许多脚注的复杂文档，使用有意义的标签

### 复杂的脚注内容

脚注可以包含 Markdown 格式：

```markdown
该算法[^algo] 是有据可查的。

[^algo]: **快速排序**算法：

    1. 选择一个枢轴
    2. 分区数组
    3. 递归排序分区
```

## 自定义标题 ID

虽然 mdPress 自动生成标题 ID，但你可以为特殊情况指定自定义 ID：

```markdown
# 我的标题 {#custom-id}

这里是一些内容。

[跳回](#custom-id)
```

自定义 ID 在以下情况下很有用：
- 你改变标题文本但想保持稳定链接
- 你需要特定 URL 结构
- 你正在从另一个文档系统迁移
- 你想要短的、易记的 ID

## 词汇表术语自动链接

mdPress 可以自动转换词汇表术语为链接。在配置中定义你的词汇表：

```yaml
# book.yaml
glossary:
  - term: "API"
    definition: "应用程序编程接口"
    link: "./glossary.md#api"
  - term: "REST"
    definition: "表述性状态转移"
    link: "./glossary.md#rest"
```

配置 `glossary` 后，每个术语的首次出现会自动链接。这在文本中显示为突出显示的术语，链接到词汇表。

### 创建词汇表章节

使用自定义标题 ID 创建词汇表章节：

```markdown
# 词汇表

## API {#api}

应用程序编程接口 - 用于构建软件应用程序的一组协议和工具。

## REST {#rest}

表述性状态转移 - 用于设计网络应用程序的架构风格。

## JSON {#json}

JavaScript 对象表示法 - 一种轻量级数据交换格式。
```

## 交叉引用

使用相对路径和片段标识符在文档之间链接：

```markdown
[查看安装指南](./installation.md)
[查阅 API 参考](./api-reference.md#authentication)
[跳转到高级选项](#advanced-options)
```

### 交叉引用最佳实践

- 使用以 `./` 或 `../` 开头的相对路径
- 链接到部分时包括片段标识符
- 在实时预览中测试链接以确保它们有效
- 避免在最终构建中链接损坏

### 程序化引用

在复杂文档中，你可能以编程方式引用部分：

```markdown
有关详情，请参见[第{{section-number}}部分](./reference.md)。
```

（注意：此示例显示该概念；实际实现取决于你的 mdPress 配置。）

## 高级功能

### 组合功能

你可以在复杂文档中组合多个扩展：

```markdown
## 算法分析 {#algorithm-analysis}

时间复杂度是 $O(n \log n)$。请查看下方的流程图[^1]：

​```mermaid
flowchart LR
    A[输入] --> B[排序]
    B --> C[输出]
​```

[^1]: 此图表使用合并排序方法。
```

### 转义特殊字符

当你需要文字美元符号或其他特殊字符时：

```markdown
价格是 \$50，不是 $50 作为公式。

转义反引号：\`文字反引号\`
```

## 故障排除

### 数学未呈现

确保美元符号未转义且 KaTeX 表达式有效。先测试简单表达式：

```markdown
测试：$x + y = z$
```

### 图表未显示

- 验证代码块语言标识符是否正确（`mermaid` 或 `plantuml`）
- 根据官方文档检查语法
- 先用更简单的图表测试
- 检查浏览器控制台的错误消息

### 脚注问题

- 确保脚注标签在引用和定义之间匹配
- 脚注定义可以在引用后的任何地方
- 保持标签名称简洁且有意义

### 自定义 ID 冲突

- ID 在文档内必须唯一
- 避免在自定义 ID 中使用空格和特殊字符
- 仅使用小写字母、数字和连字符
