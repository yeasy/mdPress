# `mdpress migrate`

[English](migrate.md)

## 作用

将 GitBook 或 HonKit 项目转换为 mdPress 格式。`migrate` 命令会检测 GitBook/HonKit 项目结构，将 `book.json` 转换为 `book.yaml`，改写 Markdown 文件中的 GitBook 特定模板标签，并输出迁移报告。

## 语法

```bash
mdpress migrate [directory] [flags]
```

## 位置参数

| 参数 | 必填 | 说明 |
| --- | --- | --- |
| `[directory]` | 否 | GitBook/HonKit 项目所在路径。省略时默认使用当前目录。 |

## 命令参数

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `--dry-run` | 关闭 | 预览会进行哪些变更但不写入文件。用于在正式迁移前检查影响范围。 |
| `--force` | 关闭 | 覆盖已有的 `book.yaml` 而不是跳过。 |

## 用法说明

### 检测

`migrate` 会在目录下查找 GitBook/HonKit 项目标记：

- `book.json`：GitBook/HonKit 配置文件
- `SUMMARY.md`：GitBook 章节结构（mdPress 已原生支持此格式）

如果两个文件都不存在，命令会报错退出。

### 转换流程

1. **读取 `book.json`**（若存在）
   - 解析 JSON 配置
   - 提取标题、作者、语言和描述
   - 转换为 mdPress 格式的 `book.yaml`
   - 添加注释头：`# Generated by mdpress migrate from book.json`

2. **保持 `SUMMARY.md` 不变**
   - mdPress 原生支持 GitBook 格式的 SUMMARY.md
   - 无需转换；此文件保持原样

3. **改写 Markdown 文件中的 GitBook 模板标签**
   - `{% hint style="TYPE" %}...{% endhint %}` → 带类型前缀的 Markdown 块引用
   - `{% code title="..." %}...{% endcode %}` → 围栏代码块（标题被删除）
   - `{% tabs %}...{% endtabs %}` 及嵌套的 `{% tab title="NAME" %}` → 四级标题（`####`）加内容

4. **递归遍历所有 `.md` 文件**
   - 跳过隐藏目录（以 `.` 开头）、`node_modules/` 和 `_book/`
   - 只处理包含已知 GitBook 标签的 `.md` 文件
   - 原地修改文件（除非指定 `--dry-run`）

5. **输出迁移报告**
   - 列出已创建的文件（如 `book.yaml`）
   - 列出已修改的文件（标签被改写的 Markdown 文件）
   - 列出已跳过的项目（如 `SUMMARY.md`）
   - 列出警告（如不支持的插件）

### 干跑模式

当指定 `--dry-run` 时：

```bash
mdpress migrate --dry-run ./my-gitbook
```

命令会打印它将要做什么，但不会修改任何文件。这对于在正式迁移前预览影响范围很有用。

### GitBook 插件警告

命令会读取 `book.json` 并警告没有对应 mdPress 等效功能的插件：

```
Warnings:
  ! GitBook plugin "disqus" has no direct mdpress equivalent — you may need to manually implement equivalent features
```

已知的原生支持或会被跳过的插件包括：
- `highlight`、`search`、`lunr`、`sharing`、`fontsettings`、`theme-default`

## 示例

```bash
# 迁移当前目录
mdpress migrate

# 迁移指定的 GitBook 项目
mdpress migrate ./my-gitbook

# 预览变更但不写入文件
mdpress migrate --dry-run ./my-gitbook

# 迁移后构建
mdpress migrate ./my-gitbook
mdpress build ./my-gitbook --format pdf
```

## 输出

### 生成的文件

当 `book.json` 存在时，命令会创建 `book.yaml`：

```yaml
# Generated by `mdpress migrate` from book.json
# Review and adjust as needed.

book:
  title: My Book
  author: John Doe
  language: en
  description: A sample book
style:
  theme: technical
output:
  filename: output
  toc: true
  cover: false
```

### 修改的文件

Markdown 文件会原地改写为标准 Markdown 等效形式：

**迁移前（GitBook）**：
```markdown
{% hint style="info" %}
这很重要。
{% endhint %}

{% code title="example.sh" %}
echo "Hello"
{% endcode %}
```

**迁移后（mdPress）**：
```markdown
> **INFO:** 这很重要。

```
echo "Hello"
```
```

## 注意事项

- 此命令最好作为**第一步**，在克隆 GitBook 项目后立即使用。
- 务必检查生成的 `book.yaml`，确保其中的元数据符合你的预期。
- 没有 mdPress 等效功能的 GitBook 插件会生成警告；你可能需要手动实现等效功能。
- 迁移后，运行 `mdpress validate` 检查结果结构。
- 运行 `mdpress build` 生成你需要的输出格式。
